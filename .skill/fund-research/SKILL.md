# 基金研究

这个 skill 面向公募基金、ETF、LOF、场内外基金及基金经理研究任务。目标不是只罗列净值涨跌，而是形成一套可复核的选基、择时、持有、调仓与退出方法论。

本 skill 有两个主信息源：

1. `efinance-cli`
2. 互联网深度搜索

当用户要求基金研究、基金筛选、基金经理分析、基金买点卖点、调仓建议、行业风格暴露分析、ETF 配置或基金组合优化时，使用本 skill。

## 强制联动规则

1. 优先触发并阅读 `efinance-cli` skill。
2. 数据应尽可能先从 `efinance-cli` 获取，并且研究过程必须实际调用 `efinance-cli` skill；只有当 `efinance-cli` 无法有效提供所需数据时，才允许用互联网补充，互联网仅作补充兜底，不得作为优先策略。
3. 若用户问题涉及最新净值阶段表现之外的信息，例如基金经理履历、经理变更、基金公告、季报、中报、年报、板块/ETF 资金流、行业政策、同类排名、销售限额、监管变化，必须进行互联网搜索。
4. `efinance-cli` 当前不擅长或未直接提供的内容，例如基金资金流、基金经理深度背景、历史公告链路、ETF 外部资金流、正式定期报告核验，必须通过互联网补齐。
5. 必须区分：
   - 已确认事实
   - 基于事实的推断
   - 尚未证实的假设
6. 分析 A 股、港股、美股相关基金时，必须补充交易额外费用拆解；场内基金至少覆盖佣金、交易规费、结算/过户费用、可能的印花税豁免或非豁免情况，场外基金至少覆盖申购费、赎回费、销售服务费、管理费、托管费及可能的持有期费率差异。
7. 当用户关注短持、轮动、定投效率、申赎节奏、套利或资金利用率时，必须显式给出资金磨损分析，说明费用对净收益和回本门槛的影响。
8. 不得把短期净值弹性误写成长期能力，也不得把单一阶段冠军误写成稳定 alpha。

## 研究目标

基金研究默认回答七个问题：

1. 选什么基金：产品本身质量如何。
2. 为什么配它：风格、赛道、策略与经理是否匹配。
3. 什么时候买：当前是否是合理建仓点。
4. 买多少：仓位与组合角色如何定。
5. 什么时候调：何时加仓、减仓、切换风格。
6. 什么时候卖：逻辑失效、风格逆风或经理变化时如何退出。
7. 成本是否值得承担：申赎费、销售服务费、场内交易费用与税费会造成多大资金磨损，是否改变持有或轮动方案。

## 默认工作流

### 阶段 1：任务定约

先明确：

- 研究对象：单只基金、基金池比较、ETF 配置、行业主题基金、宽基指数基金、主动权益、债基、固收+、FOF
- 时间框架：短线交易、波段配置、中长期配置、养老/稳健配置
- 输出目标：选基报告、买卖点建议、基金经理尽调、组合调仓建议
- 风险偏好：进攻、均衡、防守

### 阶段 2：先用 `efinance-cli` 建立底稿

优先方向：

1. 基金池与基础资料
   - `fund catalog`
   - `fund profile`
2. 净值与阶段表现
   - `fund nav history`
   - `fund nav history-batch`
   - `fund performance period`
3. 风格与持仓暴露
   - `fund allocation industry`
   - `fund allocation position`
   - `fund allocation types`
4. 经理基础信息
   - `fund managers`

默认推荐：

- `--format json`
- `--indicator-level full`
- `--trace-window 128`

但必须说明这是 skill 推荐值，不是 CLI 程序真实默认值。

若用户不希望依赖实时或环境中 `live` 不稳定，默认避免：

- `fund estimate live`

优先使用净值历史、阶段表现、配置分布和经理资料。

### 阶段 3：互联网深度搜索补齐缺口

以下信息优先通过互联网补齐：

- 基金经理详细履历、任职起止、过往管理产品、变更公告
- 基金季报、中报、年报、招募说明书、产品资料概要
- 基金公司公告、限购、清盘风险、分红、份额变动
- ETF 资金流、板块资金流、主题热度
- 同类比较、费率、跟踪误差、规模变化
- 宏观与行业因素
- 监管公示与资格底线信息

优先级：

1. 证监会、交易所、基金公司、巨潮、正式 PDF 公告
2. 基金业协会从业人员公示
3. 天天基金、东方财富等聚合页

### 阶段 4：基金基本面分析

基金的“基本面”不是企业财报，而是产品结构与策略质量。至少覆盖：

1. 产品定位
   - 主动还是被动
   - 宽基、行业、主题、债券、固收+、FOF
   - 风格是否清晰稳定

2. 业绩结构
   - 长中短期收益
   - 回撤
   - 修复能力
   - 收益是否集中在少数阶段

3. 风险收益特征
   - 波动率
   - 最大回撤
   - 收益回撤比
   - 在不同市场环境下的表现

4. 持仓与暴露
   - 行业分布
   - 仓位水平
   - 资产类型占比
   - 是否偏离宣传风格

5. 规模与可操作性
   - 规模是否过小或过大
   - 流动性是否足够
   - ETF 是否适合交易
   - 场外基金是否适合定投

6. 基金经理与团队
   - 任职区间
   - 管理年限
   - 历史产品表现
   - 是否频繁变更
   - 能力是否可迁移

### 阶段 5：技术面分析

基金技术面适合做执行层辅助，不适合替代产品研究。

#### 主动基金

重点看：

- `fund nav history`
- `fund performance period`
- 净值趋势是否平滑
- 回撤后恢复是否有力
- 是否在高位钝化或低位修复

#### ETF / 场内基金

若用户研究 ETF 或场内品种，可结合更强的技术面判断：

- 价格趋势
- 成交量
- 板块资金流
- ETF 资金流
- 重要支撑阻力
- 风格轮动位置

必要时可借助外部行情网站或公开数据补充。

### 阶段 6：买点、卖点、调仓点

#### 买点方法论

基金的买点要区分品类。

1. 宽基/核心配置基金
   - 更适合分批建仓、定投、估值带建仓
   - 不追求精确抄底
   - 重点看中期赔率与组合角色

2. 行业/主题基金
   - 先确认行业逻辑和政策/景气度
   - 再确认板块资金流与 ETF 资金流
   - 再确认净值或价格趋势未完全透支

3. 主动基金
   - 优先在风格回撤但未失效时观察建仓
   - 避免只因短期排名第一而追入
   - 要确认经理能力、风格、持仓暴露仍然一致

买点判断至少通过三个门：

- 产品定位与当前市场环境匹配
- 经理或指数逻辑未失效
- 净值/价格位置与风险收益比合理

#### 卖点方法论

卖点分四类：

1. 逻辑失效
   - 行业主线结束
   - 基金风格不再适配市场
   - 主动基金持仓与原本认知偏离
   - 指数策略失去配置价值

2. 经理变化
   - 核心经理离任
   - 团队大调整
   - 过去业绩不可再归因于现任经理

3. 估值与拥挤度过高
   - 主题过热
   - 板块资金流见顶
   - ETF 资金流放缓甚至转负
   - 净值/价格进入高风险追高区

4. 组合再平衡
   - 单一基金或单一主题权重过高
   - 与其他持仓重复暴露过多
   - 更优替代品出现

#### 调仓方法论

默认采用“目标权重 + 触发条件 + 分批执行”的方式。

加仓条件：

- 基本逻辑增强
- 风格回归占优
- 经理/策略证据提升
- 回撤后重新走强

减仓条件：

- 风险收益比下降
- 板块或风格拥挤
- 组合偏离风险预算
- 短期涨幅远超基本面支持

换仓条件：

- 同类基金里有更低费率、更强跟踪质量、更稳定经理或更清晰暴露的替代品
- 原产品风格漂移
- 原产品规模、流动性、经理稳定性变差

## 选基方法论

### 方法一：核心配置法

适用：

- 宽基 ETF
- 长期定投
- 养老与稳健配置

流程：

1. 先定核心资产类别
2. 比较费率、规模、流动性、跟踪能力
3. 用 `fund nav history` 看长期净值曲线
4. 用互联网核验产品资料概要与规模变化
5. 用分批建仓而非激进择时

### 方法二：风格/行业轮动法

适用：

- 行业主题基金
- ETF 轮动
- 中频配置

流程：

1. 互联网看行业政策、景气度、板块资金流、ETF 资金流
2. 用 `fund allocation industry` 看基金真实暴露
3. 用 `fund nav history`、`fund performance period` 看趋势与弹性
4. 不用短期冠军直接替代长期能力
5. 结合风险预算做轮动

### 方法三：主动基金经理筛选法

适用：

- 主动权益
- 偏股混合
- 固收+ 中的主动管理部分

流程：

1. 用 `fund managers` 和聚合平台建立经理候选清单
2. 搜经理变更公告与任职日期
3. 搜季报、中报、年报看持仓与策略解释
4. 比较多产品、多个市场阶段的稳定性
5. 再决定是否配置

## 基金经理尽调方法论

基金经理尽调默认采用“三层数据源”：

1. 聚合源：快速建档
   - 天天基金、东方财富等
2. 披露源：确认事实
   - 基金公司公告
   - 巨潮
   - 定期报告
   - 招募说明书
3. 监管/资格源：查底线
   - 基金业协会从业公示
   - 公开处罚或监管信息

尽调时必须回答：

- 任职区间是否清晰
- 过往业绩是否真能归因到该经理
- 多经理共管时谁主导
- 经理风格是否稳定
- 代表作是否可复制
- 是否存在重大合规或频繁变动风险

## 搜索补充方法论

### 板块/ETF 资金流

当 `efinance-cli` 不能直接给基金或板块资金流时：

1. 搜行业/概念板块资金流
2. 搜对应 ETF 资金流
3. 搜主题催化
4. 回到 `fund nav history` 与 `fund allocation industry` 验证基金是否真受益于该方向

原则：

- 板块热度只负责发现
- ETF 资金流和基金真实暴露负责确认

### 基金公告与定期报告

优先搜：

- 基金经理变更公告
- 季报、中报、年报
- 招募说明书更新
- 产品资料概要
- 限购/暂停申购赎回
- 分红、清盘风险、规模异常变化

### 同类比较

若用户要“选最优基金”，至少比较：

- 风格一致性
- 长中短期收益
- 最大回撤
- 波动
- 费率
- 规模
- 经理稳定性
- 跟踪误差或主动管理质量

## 输出要求

除非用户明确要求简版，否则默认输出应至少包含：

1. 结论摘要
2. 研究范围与日期
3. 基金定位
4. 经理与团队分析
5. 净值与阶段表现分析
6. 行业与持仓暴露分析
7. 交易额外费用拆解
8. 资金磨损分析
9. 买点判断
10. 卖点判断
11. 调仓策略
12. 风险清单
13. 事实 / 推断 / 未证实项

## 结论格式建议

建议在结尾给出一个短表：

| 项目 | 结论 |
| --- | --- |
| 结论标签 | 配置 / 观察 / 减仓 / 卖出 |
| 产品类型 | 宽基 / 行业 / 主动 / 债基 / 固收+ / ETF |
| 时间框架 | 短线 / 波段 / 中线 / 长期 |
| 主要逻辑 | 一句话 |
| 买点条件 | 触发条件 |
| 卖点条件 | 触发条件 |
| 调仓条件 | 触发条件 |
| 主要风险 | 1-3 条 |
| 置信度 | 高 / 中 / 低 |

## 重要约束

1. 不得把基金分析写成个性化理财建议。
2. 不得只看收益、不看回撤和风格来源。
3. 不得忽略基金经理变更风险。
4. 不得把披露滞后的定期报告误当成实时数据。
5. 不得省略与基金交易或持有直接相关的费用说明，尤其是场外申赎费、销售服务费和场内轮动成本。
6. 不得在需要最新信息时省略互联网核验。
7. 当证据不足时，应输出“暂不下结论”。

## 建议免责声明

`本分析仅供研究与教育用途，不构成个性化投资建议。基金产品存在净值波动、风格漂移与信息滞后风险，后续公告和市场变化可能改变判断。`
