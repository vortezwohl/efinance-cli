# 期货研究

这个 skill 面向商品期货、股指期货、国债期货及相关期货代理 ETF / 工具的中低频研究任务。目标不是只看 K 线和涨跌，而是帮助 agent 输出一套可复核、可执行、可继续跟踪的期货品种选择、方向判断、仓位管理、调仓与退出分析。

本 skill 有两个主信息源：

1. `efinance-cli`
2. 互联网深度搜索

当用户要求期货研究、期货品种筛选、方向判断、买卖点、调仓、套保思路、交割月影响、期限结构分析、宏观与产业链联动分析时，使用本 skill。

## 强制联动规则

1. 优先触发并阅读 `efinance-cli` skill。
2. 数据应尽可能先从 `efinance-cli` 获取，并且研究过程必须实际调用 `efinance-cli` skill；只有当 `efinance-cli` 无法有效提供所需数据时，才允许用互联网补充，互联网仅作补充兜底，不得作为优先策略。
3. 若问题涉及最新宏观数据、最新政策、最新库存、最新供需、最新主力合约切换、最新交割规则、最新持仓结构、最新产业事件，必须进行互联网搜索。
4. `efinance-cli` 不擅长或未直接暴露的内容，例如期限结构、基差、库存、持仓排名、产业链供需、交割规则、保证金与风控规则，必须通过互联网补齐。
5. 必须显式区分：
   - 已确认事实
   - 基于事实的推断
   - 尚未证实的假设
6. 不得把高波动行情误写成高确定性机会，也不得忽略杠杆和移仓风险。

## 研究目标

期货研究默认回答六个问题：

1. 做什么：哪个期货品种更值得研究或交易。
2. 为什么做：宏观、产业链、期限结构或资金行为是否支持方向。
3. 什么时候做：当前是否是合理开仓点。
4. 做多少：仓位、杠杆、止损预算如何定。
5. 什么时候调：何时减仓、加仓、移仓、切月或切方向。
6. 什么时候平：逻辑失效、结构变化、风控触发时如何退出。

## 默认工作流

### 阶段 1：任务定约

先明确：

- 研究对象：单一期货品种、跨品种比较、趋势交易、波段交易、对冲、套保、跨期/跨品种思路
- 品种类型：黑色、有色、能源、化工、农产品、贵金属、股指期货、国债期货
- 时间框架：日内以外的短线、波段、中线、事件驱动、交割前后
- 输出目标：研究报告、方向判断、买卖点建议、调仓策略、风控框架
- 风险偏好：保守、均衡、进攻

### 阶段 2：先用 `efinance-cli` 建立底稿

优先命令方向：

1. 品种目录与标识
   - `futures catalog`
2. 历史价格与趋势
   - `futures price history`
3. 成交与交易活跃度
   - `futures trades`

默认推荐：

- `--format json`
- `--indicator-level full`
- `--trace-window 128`

但必须说明这是 skill 推荐值，不是 CLI 程序真实默认值。

若用户不希望依赖实时或环境中 `live` 不稳定，默认避免：

- `futures price live`

优先使用目录、历史、成交类命令。

### 阶段 3：互联网深度搜索补齐缺口

以下信息优先通过互联网补齐：

- 宏观数据与政策：利率、汇率、流动性、地产、基建、出口、通胀
- 产业链供需：产量、库存、开工率、进口、出口、消费、季节性
- 期限结构：升水、贴水、近远月价差
- 基差变化
- 主力合约切换与移仓节奏
- 交易所规则、交割规则、保证金与涨跌停规则
- 持仓结构、席位变化、交易所日度周度统计
- 突发事件：天气、地缘、停产、事故、政策变化

优先级：

1. 交易所、官方统计、行业协会、监管与政策源
2. 头部产业研究与主流财经数据源
3. 聚合财经站点和市场评论

### 阶段 4：基本面分析

期货“基本面”不是企业财报，而是宏观与产业链。

至少覆盖：

1. 宏观驱动
   - 利率与流动性
   - 汇率
   - 经济周期
   - 通胀与通缩
   - 政策预期

2. 产业链供需
   - 上游供给是否收缩或扩张
   - 中下游需求是否改善或走弱
   - 库存水平是否处于拐点
   - 季节性因素是否关键

3. 结构信号
   - 基差强弱
   - 升贴水状态
   - 期限结构是否支持当前方向
   - 主力合约与次主力是否共振

4. 品种特有风险
   - 政策干预
   - 交割品变动
   - 仓单压力
   - 海外联动
   - 夜盘风险

5. 股指 / 国债期货补充
   - 股指期货看风险偏好、估值、政策、资金风格
   - 国债期货看利率预期、资金面、曲线形态、政策博弈

### 阶段 5：技术面分析

技术面是期货执行层的核心过滤器，但不能脱离基本面和结构。

至少覆盖：

1. 趋势
   - 多周期趋势方向
   - 是否突破、回踩、盘整或趋势衰竭

2. 位置
   - 当前相对近 1 个月、3 个月、6 个月区间的位置
   - 是否接近关键高低点、密集成交区、前期缺口

3. 波动
   - 波动率是否扩大
   - 是否进入高噪声环境
   - 是否超出正常风控承受范围

4. 成交与动量
   - `futures trades`
   - 放量突破还是缩量反弹
   - 价格与成交是否共振

5. 结构联动
   - 主力合约切换时走势是否失真
   - 近远月是否背离
   - 正套反套逻辑是否破坏

技术面判断必须回答：

- 现在是趋势启动区、回踩确认区、震荡混沌区、逼空/杀跌末端区，还是移仓扰动区

### 阶段 6：买点、卖点、调仓点

#### 买点方法论

1. 趋势确认型
   - 宏观或供需逻辑支持
   - 结构信号不冲突
   - 放量突破关键区间
   - 非移仓噪声驱动

2. 回踩承接型
   - 主趋势未坏
   - 回踩关键均线、前高突破位、支撑区
   - 波动收敛后重新放量

3. 事件催化型
   - 政策、库存、供给冲击、天气、地缘事件改变预期
   - 但必须防止只追单条新闻
   - 需确认期限结构和价格位置未完全透支

买点判断至少通过三个门：

- 宏观/产业逻辑没有明显逆风
- 结构信号没有明显冲突
- 风险收益比与止损位置合理

#### 卖点方法论

1. 逻辑失效
   - 宏观方向逆转
   - 供需拐点被证伪
   - 库存或政策出现相反变化

2. 技术破坏
   - 趋势线或关键支撑阻力被有效击穿
   - 放量反向运行
   - 波动失控

3. 结构恶化
   - 基差与期限结构不再支持
   - 主力移仓导致原合约失真
   - 交割风险上升

4. 组合与风控
   - 杠杆过高
   - 同方向暴露过大
   - 夜盘与事件风险超预算

#### 调仓方法论

默认采用“方向、合约、杠杆、止损、持有周期”五维调仓。

加仓条件：

- 原逻辑增强
- 趋势延续且回撤可控
- 结构信号进一步改善
- 风险预算仍有余量

减仓条件：

- 波动放大但方向不再清晰
- 结构信号转弱
- 事件兑现后赔率下降
- 临近关键数据或交割月

换仓条件：

- 主力合约切换
- 近月不再适合持有
- 其他品种的赔率或胜率更高
- 套保/对冲需求变化

## 选期货方法论

### 方法一：宏观驱动法

适用：

- 股指期货
- 国债期货
- 宏观敏感商品

流程：

1. 互联网看政策、流动性、经济周期
2. 用 `futures price history` 看趋势
3. 对照结构与事件验证
4. 再决定方向、仓位和周期

### 方法二：产业链供需法

适用：

- 黑色
- 有色
- 能源化工
- 农产品

流程：

1. 搜供给、需求、库存、季节性
2. 看期限结构与基差
3. 用 `futures price history` 验证趋势
4. 用 `futures trades` 看活跃度
5. 再决定做多、做空或观望

### 方法三：结构与移仓法

适用：

- 跨期
- 主力切换
- 交割敏感阶段

流程：

1. 先确认主力合约变化
2. 看近远月价差与移仓节奏
3. 避免把移仓噪声误当趋势
4. 必要时切换到更合适的合约观察和交易

## 搜索补充方法论

### 宏观与政策

优先搜：

- 央行、财政、产业政策
- 海外联储与美元环境
- 宏观高频数据
- 政策会议与官方表述

### 产业链与库存

优先搜：

- 行业协会与交易所数据
- 产量、库存、进口、出口
- 开工率、消费、库存天数
- 季节性和天气因素

### 交割与规则

优先搜：

- 交割规则
- 保证金调整
- 涨跌停调整
- 仓单变化
- 主力合约切换日历

## 输出要求

除非用户明确要求简版，否则默认输出应至少包含：

1. 结论摘要
2. 研究范围与日期
3. 品种与合约说明
4. 宏观与产业链分析
5. 技术面与结构分析
6. 买点判断
7. 卖点判断
8. 调仓策略
9. 风险清单
10. 事实 / 推断 / 未证实项

## 结论格式建议

建议在结尾给出一个短表：

| 项目 | 结论 |
| --- | --- |
| 结论标签 | 做多 / 做空 / 观望 / 减仓 / 平仓 |
| 品种类型 | 商品 / 股指 / 国债 / 跨期 |
| 时间框架 | 短线 / 波段 / 中线 |
| 主要逻辑 | 一句话 |
| 买点条件 | 触发条件 |
| 卖点条件 | 触发条件 |
| 调仓条件 | 触发条件 |
| 主要风险 | 1-3 条 |
| 置信度 | 高 / 中 / 低 |

## 重要约束

1. 不得把期货分析写成个性化投资建议。
2. 不得忽略杠杆、爆仓、移仓与交割风险。
3. 不得只讲技术面、不讲宏观与供需背景。
4. 不得只讲宏观逻辑、不交代执行位置与止损。
5. 不得在最新信息敏感场景下省略互联网核验。
6. 当证据不足时，应输出“暂不下结论”。

## 建议免责声明

`本分析仅供研究与教育用途，不构成个性化投资建议。期货交易具有高杠杆、高波动、移仓与交割风险，后续数据和市场事件可能快速改变判断。`
