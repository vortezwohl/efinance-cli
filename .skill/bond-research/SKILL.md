# 债券研究

这个 skill 面向利率债、信用债、可转债、债券 ETF、场内债券相关品种的中低频研究任务。目标不是只描述收益率波动，而是帮助 agent 输出一套可复核、可执行、可继续跟踪的选债、择时、持有、调仓与退出分析。

本 skill 有两个主信息源：

1. `efinance-cli`
2. 互联网深度搜索

当用户要求债券研究、债券筛选、可转债分析、债券 ETF 配置、利率与信用分析、买点卖点、调仓策略、组合防守配置时，使用本 skill。

## 强制联动规则

1. 优先触发并阅读 `efinance-cli` skill。
2. 数据应尽可能先从 `efinance-cli` 获取，并且研究过程必须实际调用 `efinance-cli` skill；只有当 `efinance-cli` 无法有效提供所需数据时，才允许用互联网补充，互联网仅作补充兜底，不得作为优先策略。
3. 若问题涉及最新利率、最新信用事件、最新评级调整、最新公告、最新政策、最新流动性环境、最新供给变化，必须进行互联网搜索。
4. `efinance-cli` 不擅长或未直接暴露的内容，例如收益率曲线、久期环境、信用评级跟踪、募集说明书细节、发行人财报、违约事件、可转债条款与强赎风险，必须通过互联网补齐。
5. 必须显式区分：
   - 已确认事实
   - 基于事实的推断
   - 尚未证实的假设
6. 分析 A 股、港股、美股债券、可转债、债券 ETF 或债基代理时，必须补充交易额外费用拆解，至少覆盖佣金、规费、结算费用、应计利息、买卖价差，以及是否存在印花税或印花税豁免。
7. 当用户关注到期收益、票息收益、波段买卖、换券、转债博弈或资金效率时，必须显式给出资金磨损分析，说明税费、价差和应计利息对实际到手收益的侵蚀。
8. 不得把短期价格波动直接当成信用改善，也不得把高票息直接当成高性价比。

## 研究目标

债券研究默认回答七个问题：

1. 买什么：债券或债券基金本身质量如何。
2. 为什么买：利率、信用、条款、转股价值或组合角色是否支持配置。
3. 什么时候买：当前是否是合理建仓点。
4. 买多少：仓位、久期和信用敞口如何定。
5. 什么时候调：何时延久期、降久期、提等级、降等级或切换券种。
6. 什么时候卖：逻辑失效、利差压缩结束、信用恶化或条款风险上升时如何退出。
7. 交易成本是否改变结论：佣金、买卖价差、应计利息与相关税费会造成多大资金磨损，是否影响配置或换券性价比。

## 默认工作流

### 阶段 1：任务定约

先明确：

- 研究对象：单只债券、债券池比较、可转债、债券 ETF、债券组合防守配置
- 债券类型：国债、政金债、城投债、产业债、可转债、短债、中长债、纯债基金代理
- 时间框架：短线、波段、中线、长期持有到期、资产配置
- 输出目标：选债报告、买卖点建议、调仓建议、风险核验
- 风险偏好：保守、均衡、进攻

### 阶段 2：先用 `efinance-cli` 建立底稿

优先命令方向：

1. 债券基础资料
   - `bond catalog`
   - `bond profile`
2. 价格与走势
   - `bond price history`
3. 资金流
   - `bond flow today`
   - `bond flow history`
4. 成交活跃度
   - `bond trades`

默认推荐：

- `--format json`
- `--indicator-level full`
- `--trace-window 128`

但必须说明这是 skill 推荐值，不是 CLI 程序真实默认值。

若用户不希望依赖实时或环境中 `live` 不稳定，默认避免：

- `bond price live`

优先使用历史、资料、日终和成交类命令。

### 阶段 3：互联网深度搜索补齐缺口

以下信息优先通过互联网补齐：

- 债券募集说明书、评级报告、评级调整公告
- 发行人财报、现金流、负债结构、担保情况
- 利率政策、央行操作、收益率曲线、信用利差
- 违约、展期、诉讼、交叉违约、再融资压力
- 可转债条款：转股价、到期赎回、回售、强赎、下修
- 债券 ETF 规模、久期、跟踪目标、申赎活跃度
- 同类券比较、期限利差、信用利差

优先级：

1. 交易所、央行、发改委、证监会、发行人公告、评级机构正式文件
2. 基金公司、债券 ETF 公开材料
3. 主流财经数据平台与新闻源

### 阶段 4：基本面分析

债券“基本面”要按类型拆开。

#### 利率债

重点看：

- 宏观增长与通胀方向
- 货币政策与流动性
- 久期环境
- 收益率曲线斜率
- 供给节奏

#### 信用债

重点看：

- 发行人盈利与现金流
- 短债与长期债到期压力
- 再融资渠道
- 资产负债结构
- 担保与增信
- 所属行业景气度
- 信用评级与评级展望

#### 可转债

重点看：

- 正股基本面
- 转股价值
- 转债溢价率
- 条款博弈
- 强赎、回售、下修风险
- 正股波动与成交活跃度

#### 债券 ETF / 债基代理

重点看：

- 投资范围
- 久期
- 杠杆
- 信用等级分布
- 跟踪偏离
- 规模与流动性

### 阶段 5：技术面分析

技术面在债券研究里是执行层工具，不是核心逻辑来源。

至少覆盖：

1. 趋势
   - 价格趋势或净值趋势
   - 近期是否进入上行、下行或盘整

2. 位置
   - 当前价格相对过去区间的高低位
   - 是否接近前高、前低或关键成交区

3. 波动
   - 债券本身是否出现异常波动
   - 是否已从稳健资产变成高波动资产

4. 量能与资金面
   - `bond flow today`
   - `bond flow history`
   - `bond trades`

5. 可转债补充
   - 正股趋势是否支持转债继续交易
   - 转债是否已经脱离正股基本面进入高拥挤区

技术面判断必须回答：

- 现在是低波稳配区、交易性博弈区、透支区，还是风险释放区

### 阶段 6：买点、卖点、调仓点

#### 买点方法论

1. 利率债买点
   - 宏观下行、流动性宽松、收益率存在下行空间
   - 久期资产赔率改善
   - 价格或净值不在明显过热区

2. 信用债买点
   - 信用风险可控
   - 信用利差仍有吸引力
   - 发行人现金流与再融资能力未恶化
   - 市场没有明显信用事件发酵

3. 可转债买点
   - 正股逻辑成立
   - 转股价值与溢价率匹配
   - 条款风险可控
   - 技术面未过热

4. 债券 ETF 买点
   - 组合需要防守或久期暴露
   - 利率环境或避险需求支持
   - 流动性与规模合格

买点判断至少通过三个门：

- 宏观或信用逻辑没有明显逆风
- 标的风险收益比合理
- 执行位置不是明显拥挤或透支区

#### 卖点方法论

1. 逻辑失效
   - 利率方向与预期相反
   - 信用恶化
   - 发行人出现重大不利事件
   - 可转债正股逻辑被证伪

2. 利差或收益率压缩结束
   - 利率债收益率下行空间已被吃完
   - 信用利差压缩后赔率明显下降

3. 技术破坏
   - 价格趋势反转
   - 资金流持续转弱
   - 成交结构恶化

4. 组合再平衡
   - 久期过长
   - 信用暴露过高
   - 单券权重过大
   - 有更高性价比替代券

#### 调仓方法论

默认采用“久期、信用、条款、流动性”四维调仓。

加仓条件：

- 利率环境更友好
- 信用利差重新打开且风险仍可控
- 可转债对应正股逻辑增强
- 组合需要提升防守或收益增强

减仓条件：

- 收益率或利差优势显著收敛
- 信用事件风险上升
- 可转债过热
- 组合久期或信用风险超预算

换仓条件：

- 同期限、更高等级或更优条款的券种出现
- ETF / 基金代理比单券更适合当前任务
- 原持仓流动性变差

## 选债方法论

### 方法一：宏观利率驱动法

适用：

- 国债
- 政金债
- 中长久期债券 ETF

流程：

1. 互联网看利率、流动性、宏观方向
2. 用 `bond price history` 看走势
3. 比较不同期限资产的弹性
4. 再决定久期配置与买点

### 方法二：信用利差筛选法

适用：

- 信用债
- 城投债
- 产业债

流程：

1. 搜发行人财报、评级、公告、信用事件
2. 用 `bond profile` 建立基础认知
3. 用 `bond price history`、`bond flow history` 看交易状态
4. 比较风险补偿是否足够
5. 再决定配置与仓位

### 方法三：可转债股债联动法

适用：

- 可转债波段
- 正股主题驱动

流程：

1. 先研究正股基本面和技术面
2. 再查转债条款、溢价率和强赎风险
3. 用 `bond price history` 看转债走势
4. 必要时同时调用股票研究方法论
5. 再决定买卖点与仓位

## 搜索补充方法论

### 利率与政策

优先搜：

- 央行操作
- 政策利率
- 资金面
- 国债收益率曲线
- 宏观增长与通胀

### 信用与发行人

优先搜：

- 评级报告与评级调整
- 发债主体财报
- 重大诉讼
- 违约、展期、交叉违约
- 再融资、担保、资产处置

### 可转债条款

优先搜：

- 募集说明书
- 强赎公告
- 回售条款
- 下修条款
- 转股价调整

## 输出要求

除非用户明确要求简版，否则默认输出应至少包含：

1. 结论摘要
2. 研究范围与日期
3. 债券类型与组合角色
4. 基本面分析
5. 技术面分析
6. 交易额外费用拆解
7. 资金磨损分析
8. 买点判断
9. 卖点判断
10. 调仓策略
11. 主要风险
12. 事实 / 推断 / 未证实项

## 结论格式建议

建议在结尾给出一个短表：

| 项目 | 结论 |
| --- | --- |
| 结论标签 | 配置 / 观察 / 减仓 / 卖出 |
| 债券类型 | 利率债 / 信用债 / 可转债 / 债券 ETF |
| 时间框架 | 短线 / 波段 / 中线 / 长期 |
| 主要逻辑 | 一句话 |
| 买点条件 | 触发条件 |
| 卖点条件 | 触发条件 |
| 调仓条件 | 触发条件 |
| 主要风险 | 1-3 条 |
| 置信度 | 高 / 中 / 低 |

## 重要约束

1. 不得把债券分析写成个性化投资建议。
2. 不得忽略信用风险、流动性风险和利率风险。
3. 不得把高票息简单等同于高性价比。
4. 不得在可转债研究中忽略正股逻辑和条款风险。
5. 不得省略与交易决策直接相关的价差、应计利息、佣金和税费说明，尤其是转债波段和换券场景。
6. 不得在需要最新信息时省略互联网核验。
7. 当证据不足时，应输出“暂不下结论”。

## 建议免责声明

`本分析仅供研究与教育用途，不构成个性化投资建议。债券与可转债存在利率、信用、流动性与条款风险，后续公告和市场变化可能改变判断。`
