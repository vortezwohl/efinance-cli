# 股票研究

这个 skill 面向以 A 股、港股、美股及 ETF 代理标的为主的中低频股票研究任务。目标不是生成空泛观点，而是让 agent 输出一套可复核、可执行、可继续跟踪的选股与交易决策分析。

本 skill 有两个主信息源：

1. `efinance-cli`
2. 互联网深度搜索

当用户要求股票研究、选股、择时、买卖点、调仓、板块轮动、资金流验证、龙虎榜解读、公司基本面与技术面综合判断时，使用本 skill。

## 强制联动规则

1. 优先触发并阅读 `efinance-cli` skill。
2. 数据应尽可能先从 `efinance-cli` 获取，并且研究过程必须实际调用 `efinance-cli` skill；只有当 `efinance-cli` 无法有效提供所需数据时，才允许用互联网补充，互联网仅作补充兜底，不得作为优先策略。
3. 若问题涉及最新行情、最新公告、最新财报、最新资金流、最新政策、最新机构观点、最新行业事件，必须进行互联网搜索。
4. `efinance-cli` 不擅长或未直接暴露的功能，例如板块资金流、ETF 资金流、深度公告核验、基金/机构背景、期权异动、监管披露，必须通过互联网补充。
5. 必须显式区分：
   - 已确认事实
   - 基于事实的推断
   - 尚未证实的假设
6. 分析 A 股、港股、美股股票或股票型 ETF 时，必须补充交易额外费用拆解，至少覆盖印花税或同类交易税、监管规费、交易所费用、过户/结算费用、券商佣金，以及这些费用是买入、卖出还是双边发生。
7. 当用户关注收益空间、波段交易、频繁调仓、短线策略或资金效率时，必须显式给出资金磨损分析，说明单次买卖或一个完整交易回合的费用侵蚀比例，避免只看价格涨跌不看实际到手收益。
8. 不得把单日波动、单篇媒体报道或单一数据源直接升级为高置信结论。

## 研究目标

使用本 skill 时，默认把股票研究拆成七个问题：

1. 买什么：候选股质量如何。
2. 为什么买：核心投资逻辑是否成立。
3. 什么时候买：买点是否合适。
4. 买多少：仓位与风险预算如何定。
5. 什么时候减：调仓与止盈逻辑是什么。
6. 什么时候卖：逻辑失效、估值透支或风险上升到什么程度必须退出。
7. 交易成本是否可接受：若在 A 股、港股、美股交易，税费、规费与佣金会造成多大资金磨损，是否改变策略性价比。

## 默认工作流

### 阶段 1：任务定约

开始前先明确：

- 研究对象：单只股票、板块内选股、指数内选股、主题轮动、事件驱动、波段交易、长期配置
- 市场范围：A 股、港股、美股、ETF
- 时间框架：短线、波段、中线、长期
- 输出目标：研究报告、候选比较、买卖点建议、调仓建议、跟踪清单
- 验证口径：尽量给出日期、指标、数据口径与来源

### 阶段 2：先用 `efinance-cli` 建立底稿

优先使用 `efinance-cli` 建立可复用底稿。默认推荐：

- `--format json`
- `--indicator-level full`
- `--trace-window 128`

但必须说明这是 skill 推荐值，不是 CLI 程序真实默认值。

优先命令方向：

1. 候选发现
   - `search`
   - `stock constituents`
2. 标的识别
   - `stock profile`
   - `stock sector`
3. 价格趋势与技术面
   - `stock price history`
   - 必要时 `stock price snapshot`
4. 资金流
   - `stock flow today`
   - `stock flow history`
5. 事件与异动
   - `stock leaderboard daily`
   - `stock performance quarterly`

如果用户明确说 `live` 不稳定或不希望依赖 `live`，默认避免：

- `stock price live`
- `market price live`

应改走历史、资料、日终和事件类命令。

### 阶段 3：互联网深度搜索补齐缺口

以下信息优先用互联网补齐：

- 板块资金流、概念资金流、ETF 资金流
- 最新财报、业绩预告、指引调整
- 公司公告、监管问询、处罚、诉讼、再融资、股权激励
- 行业政策、价格周期、供需变化
- 卖方一致预期、分析师目标价、市场分歧
- 内部人交易、回购、机构持仓变动
- 期权异动和隐含波动率背景

搜索时优先级：

1. 交易所、监管披露、公司 IR、正式公告
2. 主流金融数据平台与一级新闻源
3. 聚合型财经站点

如果信息可能变化，必须标注具体日期。

### 阶段 4：基本面分析

至少覆盖以下维度：

1. 业务理解
   - 公司靠什么赚钱
   - 收入与利润的核心驱动是什么
   - 周期、政策、价格、份额、产品结构分别影响什么

2. 财务质量
   - 营收增长
   - 毛利率、营业利润率、净利率
   - 经营现金流、自由现金流
   - 资本开支
   - 负债水平与偿债压力
   - 股东回报：分红、回购、稀释

3. 竞争格局
   - 行业增速
   - 市场份额
   - 护城河
   - 替代风险
   - 上下游议价能力

4. 估值
   - PE / PB / PS / EV/EBITDA / FCF Yield 等适配指标
   - 历史分位
   - 同业可比
   - 牛/基准/熊三情景

### 阶段 5：技术面分析

技术面不是单独决策器，而是执行层过滤器。默认从 `efinance-cli` 的历史价格与增强指标中读取。

至少覆盖：

1. 趋势
   - 长中短期趋势方向
   - 是否突破、回踩、横盘、下跌中继

2. 位置
   - 当前相对近 3 个月、6 个月、12 个月区间的位置
   - 是否接近重要前高、前低、密集成交区

3. 动量
   - 趋势是否加速或衰减
   - 强势上涨是否量价配合

4. 波动
   - 振幅是否扩大
   - 是否已经进入高波动、难以承受的交易环境

5. 量能与资金面
   - `mfi`
   - `cmf`
   - `obv`
   - `vwap`
   - `force_index`
   - `price_volume_trend`

技术面判断必须回答：

- 现在是追涨风险区、低吸观察区、突破确认区，还是趋势破坏区

### 阶段 6：买点、卖点、调仓点

#### 买点方法论

优先考虑三类买点：

1. 趋势确认型
   - 基本面逻辑成立
   - 趋势向上
   - 放量突破关键区间
   - 资金流同步改善

2. 回踩承接型
   - 主趋势未破坏
   - 回踩关键均线、前高突破位或密集成交区
   - 缩量回调、放量反包或资金流重新转强

3. 事件催化型
   - 财报、政策、产品、订单、行业催化使预期上修
   - 但必须避免只凭标题追高
   - 需确认估值与位置未完全透支

买点判断必须同时通过三个门：

- 基本面没有显著恶化
- 技术面不是明显右侧衰竭
- 风险收益比仍可接受

#### 卖点方法论

卖点至少分成四类：

1. 逻辑失效
   - 核心假设被证伪
   - 财务或业务趋势反转
   - 行业条件恶化

2. 技术破坏
   - 趋势线或关键支撑破位
   - 反弹无量
   - 资金流持续转弱

3. 估值透支
   - 股价远超基准估值
   - 催化剂兑现后缺少新驱动
   - 赔率显著下降

4. 组合优化
   - 有更高胜率或更高赔率的替代标的
   - 单一持仓权重过大
   - 板块集中度过高

#### 调仓方法论

默认采用“分批、条件式、证据驱动”的调仓原则。

加仓条件：

- 原逻辑增强而非仅价格上涨
- 新证据提升置信度
- 风险预算仍充足
- 没有出现明显过热信号

减仓条件：

- 估值扩张快于基本面改善
- 单票权重过高
- 资金流与价格背离
- 板块拥挤度显著上升

换仓条件：

- 原持仓进入低赔率阶段
- 同主题或跨主题出现更高质量标的
- 市场风格切换导致原策略失效

## 选股方法论

### 方法一：自上而下

适用场景：

- 板块轮动
- 宏观驱动
- 政策驱动
- 周期驱动

流程：

1. 互联网看宏观与行业主线
2. 看板块资金流与 ETF 资金流
3. 用 `stock sector`、`stock constituents` 找代表股
4. 用 `stock price history`、`stock flow history` 比较谁最强
5. 再做个股基本面核验

### 方法二：自下而上

适用场景：

- 长期成长
- 价值修复
- 事件驱动个股

流程：

1. 从业务、估值、财务质量出发找标的
2. 用互联网核验财报、公告、管理层信息
3. 用 `efinance-cli` 看趋势、资金流、事件面
4. 再决定买点和仓位

### 方法三：指数/篮子内择强

适用场景：

- 不想全市场扫
- 想在固定池内找强者

流程：

1. `stock constituents` 获取成分股
2. 对候选股拉 `price history`
3. 对重点票拉 `flow history`
4. 必要时用龙虎榜或公告验证
5. 按趋势、基本面、赔率综合排名

## 搜索补充方法论

### 板块与资金流

当 `efinance-cli` 不能直接给板块资金流时：

1. 搜行业/概念板块资金流
2. 搜对应 ETF 资金流
3. 搜主题龙头最近公告与催化
4. 回到 `efinance-cli` 验证龙头股趋势与个股资金流

原则：

- 板块资金流榜只负责发现
- ETF 资金流与龙头股行为负责确认

### 公司与公告

优先搜：

- 最新财报
- 业绩预告
- 问询函回复
- 回购、减持、定增、并购
- 诉讼、监管处罚

### 技术面外部补充

如果用户需要更强的技术面背景，可搜索：

- 期权成交量与隐含波动率
- 机构目标价变化
- 行业价格指数与供需数据

## 输出要求

除非用户明确要求简版，否则默认输出应至少包含：

1. 结论摘要
2. 研究范围与日期
3. 核心投资逻辑
4. 基本面分析
5. 技术面分析
6. 交易额外费用拆解
7. 资金磨损分析
8. 买点判断
9. 卖点判断
10. 调仓策略
11. 风险清单
12. 事实 / 推断 / 未证实项

## 结论格式建议

建议在结尾给出一个短表：

| 项目 | 结论 |
| --- | --- |
| 结论标签 | 买入 / 观察 / 减仓 / 卖出 |
| 时间框架 | 短线 / 波段 / 中线 / 长期 |
| 主要逻辑 | 一句话 |
| 买点条件 | 触发条件 |
| 卖点条件 | 触发条件 |
| 调仓条件 | 触发条件 |
| 主要风险 | 1-3 条 |
| 置信度 | 高 / 中 / 低 |

## 重要约束

1. 不得把内容写成个性化投资建议。
2. 不得只给买入观点、不写卖出与失效条件。
3. 不得只讲基本面、不讲执行层买卖点。
4. 不得只讲技术面、不交代业务与估值约束。
5. 不得省略与交易决策直接相关的税费、规费、佣金和资金磨损说明，尤其是短线、波段和高换手场景。
6. 不得在最新信息敏感场景下拒绝联网核验。
7. 当证据不足时，应输出“暂不下结论”而不是伪造确定性。
